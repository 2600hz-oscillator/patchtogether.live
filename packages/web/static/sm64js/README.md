# /sm64js — served at runtime

Pre-built sm64js webpack bundle, served by SvelteKit at `/sm64js/*`.

- `sm64js.bundle.js` — ~12 MB minified ES5 bundle of the sm64js engine
  (LFS-tracked; see repo root `.gitattributes`). Loaded by the SM64
  module's card as `<script src="/sm64js/sm64js.bundle.js">` once the
  required DOM scaffold has been injected. Exposes the produce-one-frame
  step entry, the `WebGLInstance`, and `playerInput` on `window`.
- `sm64js.bundle.LICENSE.txt` — License banners extracted by webpack
  (third-party MIT/BSD bundled deps; idb-keyval, msgpack-lite, ...).
  Kept alongside the bundle so anyone downloading it has the full
  attribution chain.
- `template.css` — the upstream's `template.css`, served verbatim.
  Linked by `Sm64Card.svelte` *only* when an SM64 instance is mounted
  (so we don't leak Bootstrap-y selectors into the rest of the app).

The bundle is regenerated by re-running upstream's `npm run build`; see
`packages/web/native/sm64js/README.md` for the exact recipe + upstream
commit pin.
